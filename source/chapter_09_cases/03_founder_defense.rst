--------------
创始人防御策略
--------------

创始人如何在多轮融资后仍牢牢掌握公司控制权，是公司治理中最核心也最现实的课题。
本章系统梳理六种主要防御策略，并结合真实案例说明其适用场景与局限性。

AB股防御
========

AB股（双层股权结构）是最经典的创始人防御工具——将低投票权的B股公开发行，
而创始人持有高投票权的A股（通常1股对应10-20票），从而以少数股权控制多数投票权。

::

   以京东为例：
     刘强东持股约12%，但通过AB股控制约70%投票权
     Amazon普通股：1股=1票
     京东A类股：1股=1票
     京东B类股：1股=20票
     → 创始人12%的股权，控制了70%以上的投票权
     → 即使股权被稀释到个位数，投票权依然稳如磐石

.. list-table:: AB股案例对比
   :header-rows: 1
   :widths: 20 25 25 30

   * - 公司
     - 创始人
     - 股权比例
     - 投票权比例
   * - 京东
     - 刘强东
     - ~12%
     - ~70%（B股1:20）
   * - 小米
     - 雷军
     - ~31%
     - ~55%（B股1:10）
   * - 美团
     - 王兴
     - ~11%
     - ~60%（B股1:10）

注意：AB股并非万能。香港联交所在2018年才允许同股不同权公司上市，
且要求"日落条款"——创始人离世或失去行为能力后，
高投票权股自动转为普通股。

GP-LP架构防御
=============

通过有限合伙（Limited Partnership）架构，创始人以普通合伙人（GP）身份控制持股平台，
即使出资金额很小，也能行使平台的完整投票权。

::

   典型架构：
     创始人 → GP（普通合伙人，出资0.1%-1%）
     员工/投资人 → LP（有限合伙人，出资99%-99.9%）
     → GP以极小出资控制平台全部投票权
     → LP只享有经济收益，不参与投票决策

实战案例——多家AI芯片公司采用GP架构锁定控制权：

::

   案例：某AI芯片创业公司
     创始人团队 GP控制 持股平台 → 持有公司30%股份
     投资人 LP 参与平台 → 享有分红权但无投票权
     创始人以0.5%的平台出资，控制了30%的公司投票权

GP-LP架构的优势在于隐蔽性强——外界看到的只是"某持股平台持有30%"，
但内部投票权实际由GP牢牢掌控。这是AB股之外最常用的防御工具。

一致行动人协议
==============

一致行动人协议是创始人联合其他股东，通过书面协议约定在股东会上统一投票，
从而形成稳定的投票同盟。

::

   协议核心条款：
     - 各方在股东会/董事会投票前须协商一致
     - 无法达成一致时，以创始人意见为准
     - 协议期限通常3-5年，到期自动续期
     - 违约方须承担高额违约金

实战案例：

::

   某科技公司创始团队通过一致行动人协议守住控制权：
     创始人A持股15%
     联合创始人B持股8%
     联合创始人C持股7%
     三名创始人签署一致行动人协议 → 合计30%投票权
     投资人虽然单家持股18%，但无法撼动创始人联盟

一致行动人协议的弱点在于信任基础——一旦联盟内部出现裂痕，
协议可能被撕毁。因此实践中常与AB股或GP架构搭配使用。

公司章程防御条款
================

在公司章程中嵌入创始人保护条款，是防止"制度性突袭"的关键防线。

.. list-table:: 常见章程防御条款
   :header-rows: 1
   :widths: 30 70

   * - 条款类型
     - 说明
   * - 董事提名权
     - 创始人有权提名特定数量的董事候选人，提名须列入股东会议案
   * - CEO任免须创始人同意
     - 董事会有权聘任/解聘CEO，但须经创始人书面同意
   * - 超级多数条款
     - 重大事项（合并、解散、章程修改）须2/3或3/4以上股东同意
   * - 优先购买权/优先认购权
     - 创始人有权在新股发行时按比例认购，防止股权被稀释

::

   重要提示：公司章程防御条款需要在**首轮融资之前**写入章程。
   一旦投资人进入，创始人再想修改章程增加保护条款将极为困难，
   因为章程修改本身就需要股东会（含投资人）的多数同意。

董事会席位控制
==============

控制董事会比控制股东会更直接——因为日常经营决策由董事会做出，
CEO任免也由董事会决定。

::

   核心策略：融资时约定创始人占董事会多数席位
     董事会5席：创始人3席 + 投资人2席
     董事会7席：创始人4席 + 投资人2席 + 独董1席
     → 创始人永远占董事会多数
     → 没有创始人同意，无法通过任何重大决议

.. list-table:: 董事会控制的关键条款
   :header-rows: 1
   :widths: 30 70

   * - 条款
     - 说明
   * - 董事会组成
     - 创始人团队在董事会中始终保持多数席位（如5席中占3席）
   * - 董事罢免限制
     - 投资人罢免创始人董事须经创始人同意
   * - 董事会空缺填补
     - 董事辞职或离任后，空缺由创始人提名填补
   * - 委员会组成
     - 审计委员会、薪酬委员会等关键委员会须有创始人代表

从实践来看，董事会席位控制是最有效的防御手段之一——即使投资人股份占多数，
只要创始人控制董事会，就不会被"程序性逐出"。

多层防御体系
============

单一防御工具都有漏洞，成熟的创始人防御策略是**多层叠加**。

::

   建议的防御层级（从外到内）：
     第一层：AB股/同股不同权 —— 控制股东会投票权
     第二层：一致行动人协议 —— 巩固创业团队投票联盟
     第三层：GP-LP架构 —— 控制员工持股平台投票权
     第四层：公司章程条款 —— 董事提名权+CEO任免否决权
     第五层：董事会席位控制 —— 创始人在董事会占多数
     第六层：一票否决权限制 —— 限制投资人的一票否决权范围

::

   实战案例——某成功上市的SaaS公司防御体系：
     A轮融资（2018）：
       创始人要求：董事会5席中创始人占3席 ✅
       章程写入：CEO任免须创始人同意 ✅
       投资人一票否决权范围限制在：增资、分红、清算 ✅

     B轮融资（2020）：
       保持董事会多数 ✅
       增设GP架构控制员工持股平台 ✅
       所有投资人签署一致行动人协议，以创始人意见为准 ✅

     IPO（2023）：
       设置AB股，创始人1股=10票 ✅
       三层防御全部就位，创始人持股15%控制65%投票权 ✅

注意，防御体系不是越多越好。过度防御会让投资人望而却步，
关键在于找到创始人控制与投资人权益之间的平衡点。
对于早期创业者，建议先锁定**董事会多数 + 一票否决权限制**这两条底线，
后续融资中逐步补充AB股和GP架构。
